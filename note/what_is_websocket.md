意思就是，若採用一般通訊協議，我們是無法正常使用聊天功能。

因為聊天功能是伺服器會主動傳送對方的訊息給我。

而此時我還沒有發送請求伺服器給我訊息。

這就是差異。

簡單來說，WebSocket 就像是一條電話線，讓你的瀏覽器和伺服器之間可以直接對話。傳統的網頁通訊方式（HTTP）就像是你每次想問朋友一個問題，都得掛斷電話再重撥一次，這樣效率很低。而 WebSocket 則讓你們可以一直保持通話，隨時互相傳送訊息，這樣就能即時交流，不需要反覆建立連線。

**WebSocket 的優點**

- **即時互動**：伺服器可以立即把最新消息傳給你的瀏覽器，不需要等你主動詢問，像是即時聊天或線上遊戲。

- **節省資源**：因為連線一直保持著，不需要每次都重新連線，減少了不必要的資料傳輸，節省頻寬。

**WebSocket 的運作方式**

1. **建立連線**：你的瀏覽器向伺服器發出請求，說「我們改用 WebSocket 來通訊吧」。

2. **確認連線**：伺服器同意後，連線就建立起來，雙方可以開始自由對話。

3. **傳輸資料**：在這條「電話線」上，雙方可以隨時傳送訊息，不需要等待對方先說話。

**應用場景**

- **即時聊天**：像是線上客服或聊天室，訊息能立即送達。

- **即時通知**：像是股票價格變動，讓你隨時掌握最新資訊。

- **多人遊戲**：玩家之間的動作能即時同步，提升遊戲體驗。

總之，WebSocket 讓網頁應用程式能夠更快速、即時地與伺服器溝通，帶來更流暢的使用者體驗。 

即時聊天應用程式需要在使用者之間快速傳遞訊息，這要求伺服器能夠立即將新訊息推送給所有相關的客戶端。傳統的 HTTP 協定是單向的，客戶端必須主動發送請求才能獲取資料，伺服器無法主動將資料推送給客戶端。這意味著，若使用 HTTP，客戶端需要不斷地向伺服器發送請求（稱為「輪詢」）以檢查是否有新訊息，這樣會導致伺服器負載增加，且無法實現真正的即時通訊。 

相較之下，WebSocket 協定允許建立持久的雙向連接，伺服器可以主動將新訊息即時推送給客戶端，實現真正的即時通訊，並減少不必要的網路流量和伺服器負載。 

因此，為了實現高效的即時聊天功能，使用 WebSocket 協定比傳統的 HTTP 協定更為適合。
